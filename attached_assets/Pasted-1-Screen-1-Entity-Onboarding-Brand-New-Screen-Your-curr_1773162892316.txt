2. Screen 2: "Intelligent Data Ingestion & Classification" (Update Slide 11)

You need to overhaul your “Ingestion” page to show the new document types and the Human-in-the-Loop feature.

Update the Document Names:
Remove the old GST/ITR labels. Your drag-and-drop zone must explicitly show the 5 new required documents:

• ALM (Asset-Liability Management)
• Shareholding Pattern
• Borrowing Profile
• Annual Reports
• Portfolio Cuts / Performance Data

The "Human-in-the-Loop" Feature (CRITICAL):
Do not just show “Done” next to uploaded files. Instead show an Auto-Classification Label with approval buttons.

Example UI:
File uploaded → AI tags it as [Detected: Shareholding Pattern]

Next to this show:
• Green "Approve" (✓) button
• Grey "Edit/Deny" (✎) dropdown

Why this stands out:
The judges explicitly asked for “users to approve, deny, or edit the auto-classification”. This UI widget directly answers that requirement.

3. Screen 3: "Dynamic Schema Mapping" (New Modal / Tab)

The brief asks for a way to let users “define or configure the output schema.” This is the core technical challenge.

The Interface:
Add a prominent button on the ingestion screen:

⚙️ Configure Extraction Schema

What it looks like (when clicked):
Open a modal or slide-over panel showing a visual mapping tool.

Left Side – Raw AI Extracted Data
• Raw_Profit_2023
• Raw_Revenue_2023

Right Side – Bank's Standard Schema (Dropdown Mapping)
• Raw_Profit_2023 → EBITDA_Current_Year
• Raw_Revenue_2023 → Revenue_Current_Year

Why this stands out:
Most teams will ignore this because it sounds complex. Showing a Schema Configurator UI mockup proves you have thought through enterprise-level data architecture.
